## 新增需求

### 需求：主题系统实现
系统**必须**支持三种主题模式（日间/夜间/跟随系统），通过 CSS 变量和 `data-theme` 属性切换，持久化到 localStorage。

#### 场景：主题切换
- **当** 用户在导航栏点击主题切换按钮
- **那么** 页面主题在日间/夜间/跟随系统之间循环切换
- **并且** 所有 CSS 变量（surface、content、border、glass 等）同步更新
- **并且** 选择持久化到 localStorage，刷新后保持

#### 场景：跟随系统模式
- **当** 用户选择"跟随系统"主题
- **并且** 操作系统切换明暗模式
- **那么** 页面自动跟随系统设置变化

#### 场景：CSS 变量完整性
- **当** 主题切换时
- **那么** 以下 CSS 变量必须全部正确映射：表面色（4级）、内容色（4级）、边框色（2级）、毛玻璃（3项）、卡片（3项）、输入框（3项）、遮罩（1项）

### 需求：语义化颜色 Token
Tailwind 配置**必须**扩展语义化颜色（surface、content、border），通过 CSS 变量映射实现主题感知。

#### 场景：组件使用语义色
- **当** 开发者使用 `bg-surface` 或 `text-content-secondary` 类名
- **那么** 颜色自动跟随当前主题变化，无需手动区分明暗模式
