## 新增需求

### 需求：语音识别页面（Whisper）
系统**必须**提供 Whisper 语音识别页面（`/speech`），支持录音、提交音频、显示识别结果和发音评分。

#### 场景：录音操作
- **当** 用户点击"开始录音"按钮
- **那么** 请求麦克风权限并开始录音
- **并且** 录音按钮变为红色脉冲动画
- **当** 用户点击"停止录音"
- **那么** 停止录音，准备提交

#### 场景：提交语音识别
- **当** 用户有录音数据并点击识别
- **那么** POST 请求 `/whisper/recognize`（multipart/form-data）
  - 参数：`audioFile`（录音文件）、`word`（目标单词）、`type`（识别类型）
- **当** 返回成功
- **那么** 显示识别结果 `result`（是否匹配）和 `scoring`（发音评分）

#### 场景：发音评分展示
- **当** 有评分结果
- **那么** 以环形进度条展示分数
- **并且** 高分（≥80）绿色，中等（60-79）黄色，低分（<60）红色

### 需求：Vosk 本地语音识别页面
系统**必须**提供 Vosk 本地离线语音识别页面（`/speech/vosk`），功能类似 Whisper 页面但使用本地模型。

#### 场景：Vosk 识别
- **当** 用户录音并提交
- **那么** 使用本地 Vosk 模型进行离线识别
- **并且** 显示识别结果

**交叉引用**：依赖 `frontend-data-layer`、`frontend-ui-components`（Progress 环形进度条）、`frontend-layout`
