## 新增需求

### 需求：单词列表页
系统**必须**提供单词卡片网格浏览页面（`/words`），支持分页、状态筛选（全部/未学/已学/收藏）、搜索，以卡片网格展示单词。

#### 场景：单词列表加载
- **当** 用户进入单词列表页
- **那么** 调用后端获取单词列表（需要新增 JSON API 替代原 PartialView `/word/learnenglishlist`）
- **并且** 以响应式网格展示单词卡片（2列→3列→4列→5列→6列）

#### 场景：状态筛选
- **当** 用户切换 Tab（全部/未学/已学/收藏）
- **那么** 对应调用 `zt=1`（全部）、`zt=2`（未学）、`zt=3`（已学）或收藏列表 API
- **并且** 列表平滑切换，先显示骨架屏再显示数据

#### 场景：搜索单词
- **当** 用户在搜索框输入关键词
- **那么** 带 `name=xxx` 参数请求过滤后的列表

#### 场景：分页
- **当** 单词总数超过每页数量
- **那么** 底部显示分页器（页码导航）

### 需求：单词卡片交互
单词卡片**必须**按照 DESIGN_SYSTEM.md 定义的 WordCard 规范渲染，支持悬浮效果、发音播放、收藏操作。

#### 场景：发音播放
- **当** 用户点击单词卡片上的发音按钮
- **那么** 播放该单词的英/美发音音频

#### 场景：收藏/取消收藏
- **当** 用户点击收藏按钮
- **那么** POST 请求 `/word/setcollect?lexiconId=xxx&isCollect=true/false`
- **并且** 心形图标 bounce-in 动画 + 颜色变化

#### 场景：设置学习状态
- **当** 用户点击标记已学/未学
- **那么** POST 请求 `/word/szzt?zt=xxx&dqzt=xxx&lexiconId=xxx`

### 需求：单词表格视图
系统**必须**提供单词表格展示视图（`/words/table/:courseId`），以表格形式展示课程内的单词。

#### 场景：表格展示
- **当** 用户选择表格视图
- **那么** 显示数据表格（单词、音标、释义、状态、操作列）
- **并且** 行 hover 高亮

### 需求：单词详情页
系统**必须**提供单词详情页面（`/words/:word`），展示单词的完整信息（释义、音标、例句、发音）。

#### 场景：单词详情加载
- **当** 用户点击单词卡片进入详情
- **那么** 调用 `GET /translate/xzdc?content=xxx` 获取有道词典详情
- **并且** 调用 `GET /word/wordexist?kc=xxx&en=xxx` 检查发音音频可用性

#### 场景：编辑单词释义
- **当** 用户编辑单词释义并保存
- **那么** POST 请求 `/word/updc?id=xxx&en=xxx&cn=xxx`

#### 场景：删除单词
- **当** 用户确认删除单词
- **那么** POST 请求 `/word/deletedc?coursecontentId=xxx`

### 需求：学习状态校准
系统**必须**提供一键校准学习状态的功能。

#### 场景：校准操作
- **当** 用户点击"校准"按钮
- **那么** POST 请求 `/word/calibration`
- **并且** 成功后刷新列表数据

**交叉引用**：依赖 `frontend-data-layer`、`frontend-ui-components`（WordCard/Table/Skeleton/Pagination）、`frontend-layout`
