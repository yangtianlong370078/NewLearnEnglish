## 新增需求

### 需求：完整动画体系
系统**必须**实现 DESIGN_SYSTEM.md 中定义的 20+ 种关键帧动画和微交互，并遵循 `prefers-reduced-motion` 媒体查询。

#### 场景：入场动画
- **当** 组件首次渲染
- **那么** 支持 fade-in、fade-in-up、fade-in-down、fade-in-left、fade-in-right、scale-in、slide-in-right、slide-in-bottom 入场动画

#### 场景：持续动画
- **当** 元素需要持续吸引注意力
- **那么** 支持 float（悬浮）、pulse-soft（脉冲）、shimmer（闪烁）、gradient-shift（渐变流动）、spin-slow（慢旋转）、waveform（波形）等持续动画

#### 场景：特殊动画
- **当** 触发特殊交互
- **那么** 支持 flip（翻牌）、confetti（庆祝烟花）、count-up（数字滚动）、typing（打字机）动画

#### 场景：减少动画偏好
- **当** 用户操作系统设置 `prefers-reduced-motion: reduce`
- **那么** 所有动画时长降为 0.001ms，不影响功能

### 需求：滚动入场动画
系统**必须**使用 Intersection Observer 实现区块滚动到视口时的入场动画。

#### 场景：区块滚动入场
- **当** 页面区块滚动进入视口（阈值 10%）
- **那么** 触发 fade-in-up 动画
- **并且** 动画仅触发一次

### 需求：响应式设计
系统**必须**实现移动端优先的响应式布局，覆盖 6 个断点（默认/<640/≥1024/≥1280/≥1536）。

#### 场景：移动端自适应
- **当** 屏幕宽度 < 640px
- **那么** 单词卡片 2 列、页面单列、底部 Tab 栏可见、侧边栏隐藏

#### 场景：平板自适应
- **当** 屏幕宽度 768px - 1023px
- **那么** 单词卡片 4 列、侧边栏抽屉模式

#### 场景：桌面端自适应
- **当** 屏幕宽度 ≥ 1024px
- **那么** 单词卡片 5-6 列、侧边栏固定、底部 Tab 隐藏

#### 场景：触控优化
- **当** 使用触控设备
- **那么** 所有可点击元素最小触控区域 44×44px
- **并且** 列表项最小高度 48px

**交叉引用**：依赖 `frontend-theme`（主题变量）、`frontend-ui-components`（所有组件的动画实现）
